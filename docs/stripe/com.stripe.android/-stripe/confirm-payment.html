Confirm and, if necessary, authenticate a PaymentIntent. Used for automatic confirmation flow.

For confirmation attempts that require 3DS1 authentication, Custom Tabs will be used to render the 3DS1 page if both of the following are true:

  • Custom Tabs are supported on the device

  • The return_url in the confirmation request is not set (i.e. set to null)

Otherwise, a WebView will be used.

No return_urlCustom TabsWebView
Custom return_urlWebViewWebView

Parameters

activity

the Activity that is launching the payment authentication flow

confirmPaymentIntentParams

ConfirmPaymentIntentParams used to confirm the PaymentIntent

stripeAccountId

Optional, the Connect account to associate with this request. By default, will use the Connect account that was used to instantiate the Stripe object, if specified.
