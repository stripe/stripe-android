Properties

amount

 val amount: Long?

A positive integer in the smallest currency unit (that is, 100 cents for $1.00, or 1 for ¥1, Japanese Yen being a zero-decimal currency) representing the total amount associated with the source. This is the amount for which the source will be chargeable once ready. Required for single_use sources.

clientSecret

 val clientSecret: String?

The client secret of the source. Used for client-side retrieval using a publishable key.

codeVerification

 val codeVerification: SourceCodeVerification?

Information related to the code verification flow. Present if the source is authenticated by a verification code (flow is code_verification).

created

 val created: Long?

Time at which the object was created. Measured in seconds since the Unix epoch.

currency

 val currency: String?

Three-letter ISO code for the currency associated with the source. This is the currency for which the source will be chargeable once ready. Required for single_use sources.

flow

 val flow: String?

The authentication flow of the source. flow is one of redirect, receiver, code_verification, none.

id

 val id: String?

Unique identifier for the object.

isLiveMode

 val isLiveMode: Boolean?

Has the value true if the object exists in live mode or the value false if the object exists in test mode.

metaData

 val metaData: Map<String, String>?

Set of key-value pairs that you can attach to an object. This can be useful for storing additional information about the object in a structured format.

owner

 val owner: SourceOwner?

Information about the owner of the payment instrument that may be used or required by particular source types.

receiver

 val receiver: SourceReceiver?

Information related to the receiver flow. Present if the source is a receiver (flow is SourceFlow.RECEIVER).

redirect

 val redirect: SourceRedirect?

Information related to the redirect flow. Present if the source is authenticated by a redirect (flow is SourceFlow.REDIRECT).

sourceOrder

 val sourceOrder: SourceOrder?

Information about the items and shipping associated with the source. Required for transactional credit (for example Klarna) sources before you can charge it.

sourceTypeData

 val sourceTypeData: Map<String, Any?>?

 val sourceTypeModel: SourceTypeModel?

statementDescriptor

 val statementDescriptor: String?

Extra information about a source. This will appear on your customer’s statement every time you charge the source.

status

 val status: String?

The status of the source, one of canceled, chargeable, consumed, failed, or pending. Only chargeable sources can be used to create a charge.

type

 val type: String

Gets the SourceType of this Source, as one of the enumerated values. If a custom source type has been created, this returns SourceType.UNKNOWN. To get the raw value of an SourceType.UNKNOWN type, use typeRaw.

typeRaw

 val typeRaw: String

Gets the type of this source as a String. If it is a known type, this will return a string equal to the SourceType returned from type. This method is not restricted to known types

usage

 val usage: String?

Either reusable or single_use. Whether this source should be reusable or not. Some source types may or may not be reusable by construction, while others may leave the option at creation. If an incompatible value is passed, an error will be returned.
